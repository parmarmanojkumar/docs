### Description

Noma Security is a unified platform to secure and govern AI agents and autonomous systems. It delivers safety, security, and compliance controls at the runtime layer — evaluating and enforcing policy on every agent action before execution occurs. Noma was the first company to achieve AARM Core conformance, completing the full testing protocol against R1–R6 of the specification.

### Company

Noma Security was founded to address the enforcement gap that exists between AI agents and the systems they interact with — a layer that traditional security tooling was never designed to protect. The platform is built for organizations deploying AI agents at scale in production environments where auditability, policy control, and runtime visibility are non-negotiable.

### Use cases

  <Accordion title="Enterprise AI deployments">

### Architecture

Noma operates as an enforcement proxy between the agent runtime and downstream tool targets. Each action request is intercepted and evaluated against a policy engine before any side effect occurs. The system maintains session-scoped context to support multi-step reasoning about accumulated intent — satisfying the AARM R3 context accumulation requirement.

The enforcement decision returns one of five outcomes defined in the AARM specification: `ALLOW`, `BLOCK`, `DEFER`, `REDACT`, or `ESCALATE`. All decisions are written to an immutable audit log before the action result is returned to the agent.
