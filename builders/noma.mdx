### Description

Noma Security is a unified platform to secure and govern AI agents and autonomous systems. It delivers safety, security, and compliance controls at the runtime layer — evaluating and enforcing policy on every agent action before execution occurs. Noma was the first company to achieve AARM Core conformance, completing the full testing protocol against R1–R6 of the specification.

Noma Security was founded to address the enforcement gap that exists between AI agents and the systems they interact with — a layer that traditional security tooling was never designed to protect. ### Architecture

Noma operates as an enforcement proxy between the agent runtime and downstream tool targets. Each action request is intercepted and evaluated against a policy engine before any side effect occurs. The system maintains session-scoped context to support multi-step reasoning about accumulated intent — satisfying the AARM R3 context accumulation requirement.

The enforcement decision returns one of five outcomes defined in the AARM specification: `ALLOW`, `BLOCK`, `DEFER`, `REDACT`, or `ESCALATE`. All decisions are written to an immutable audit log before the action result is returned to the agent.

### Core requirements (R1–R6)

<Steps>
  <Step title="R1 — Action interception">
    The system intercepts agent action requests before any side effect occurs. Noma operates as a proxy that receives every tool call prior to forwarding.
  </Step>
  <Step title="R2 — Policy-based authorization">
    Each action is evaluated against a configurable policy set. The system returns an authorization decision before execution proceeds.
  </Step>
  <Step title="R3 — Context accumulation">
    The system maintains session-scoped context across the full agent interaction. Prior actions are available to the policy engine when evaluating subsequent requests.
  </Step>
  <Step title="R4 — Five authorization decisions">
    The system supports all five AARM authorization outcomes: ALLOW, BLOCK, DEFER, REDACT, and ESCALATE.
  </Step>
  <Step title="R5 — Immutable audit log">
    All authorization decisions and their inputs are written to a tamper-evident log before the action result is returned to the agent.
  </Step>
  <Step title="R6 — AARM system designation">
    Noma refers to its product as an AARM system, maintains a dedicated page explaining conformance, and has active production customers with verified security posture.
  </Step>
</Steps>

### Deploying Noma as your AARM system

Noma can be deployed as a conformant AARM enforcement layer for any agent stack. The integration sits between your agent runtime and downstream tools or APIs — no changes to agent code are required.

<Steps>
  <Step title="Route agent traffic through Noma">
    Configure your agent framework to proxy tool calls through the Noma enforcement endpoint. Noma supports major agent frameworks and MCP-compatible runtimes.
  </Step>
  <Step title="Define your policy set">
    Write ALLOW / BLOCK / DEFER rules that reflect your organization's risk posture and compliance requirements. Noma provides a policy editor and a library of baseline templates for common use cases.
  </Step>
  <Step title="Enable audit logging">
    Connect your SIEM or log aggregator to receive the AARM-compliant audit stream. Noma emits structured decision records for every action evaluated.
  </Step>
  <Step title="Reference your AARM system">
    Document Noma as your AARM system in security documentation, vendor attestations, and compliance evidence packages. The conformance report can be attached directly to audit artifacts.
  </Step>
</Steps>
