### Description

Formal is a protocol-aware reverse proxy that enforces least privilege at the wire-protocol level across data, infrastructure, and AI agent traffic. It sits between identities (humans and AI agents) and resources like databases, warehouses, SSH/Kubernetes servers, and MCP servers, parsing wire protocols natively and evaluating security policies inline on every request.

For AI agent workloads, Formal proxies traffic between agents and resources, applying the same policy surface it applies to human-initiated database queries: identity resolution, query-level authorization, PII masking, tool-call filtering, and full audit capture. Because the proxy parses wire protocols natively rather than treating it as opaque TCP traffic, it can make decisions on the specific tool being invoked, the arguments being passed, and the data flowing back.

Formal's relevance to AARM is direct: it operates at the wire-protocol chokepoint where agents call tools, which makes it a natural pre-execution enforcement point for authorization, identity binding, and tamper-evident auditing of autonomous agent activity.

### Architecture

Formal enforces AARM requirements through interception at two complementary layers. The first is a client-side layer that sits between a developer's AI coding tools and the model APIs they call, allowing agent tool calls to be inspected and blocked pre-execution based on policy. Customers who want observability only can operate without this layer; customers who want enforcement route traffic through it.

The second layer is Formal's protocol-aware proxy between identities and downstream resources such as databases, HTTP services, and productivity suites. It applies policies across session, request, and response stages and can allow, block, or modify, a given request. Because agent-originated traffic carries context from the first layer, the proxy can differentiate a human-issued query from one issued by an agent session and apply controls accordingly.

Session context is captured across both layers, with the agent session as the parent and downstream requests recorded as linked child events. This gives operators a full replay of every action an agent took, scoped to the resolved identity of the user whose environment initiated it. Policy decisions, identity bindings, and tool calls are written to a tamper-evident audit trail, exportable to common SIEM and observability backends.

Policy evaluation today is deterministic and pre-execution only. Formal's policy surface is limited to static scenarios that block or modify agent actions before they execute based on predefined rules, rather than evaluating non-deterministic intent alignment at runtime. Asynchronous behavioral analysis runs alongside the enforcement path and is on the roadmap to feed into real-time policy inputs.

### Company

Formal is building the protocol-aware security proxy layer for data, infrastructure, and AI agent workloads. Rather than operating at the network layer like a traditional firewall, Formal parses wire protocols natively which lets it make query-level decisions on who is doing what to which resource, in real time.

The company's thesis is that as AI agents proliferate inside enterprises, the enforcement point that matters most is the one closest to the resource, not the one closest to the agent. A proxy that understands the wire protocol can strip PII from responses before they reach an LLM, block unauthorized tool calls, and enforce permissions that the upstream system does not natively support, without requiring any changes to the application, SDKs, or to the agent itself.

Formal's AI agent use case sits alongside its original data access governance product, and the same proxy serves both: the policy engine, audit layer, and deployment model are unified across human and agent traffic. Customers include Notion, Cursor, Anchorage, and EliseAI.
